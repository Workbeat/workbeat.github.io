/movimientosPersonal get post

get

Obtener la información básica de todos los movimientos de personal existentes.

post

Crear un movimiento de personal.

Es requerido tener el permiso para crear movimientos de personal.

Se valida:

  • Que exista el empleado para el cual se quiere crear un movimiento de personal.
  • Que el tipo de movimiento de personal sea válido.
  • Que la fecha de aplicación del movimiento sea válida (por ejemplo, si es una baja, la fecha debe ser posterior a la fecha de ingreso del empleado).
  • Que la nueva posición a asignar corresponda con el tipo de movimiento de personal (por ejemplo, si se indica que se trata de una transferencia, la nueva posición debe pertenecer a otra organización o bien ser diferente a la posición actual en alguno de sus atributos).
  • Que la nueva posición tenga habilitado el tipo de plantilla que se quiere asignar al empleado.
  • Si se trata de una baja, que se especifique el motivo de baja del empleado (consultar catálogo de motivos de baja).
  • Si se trata de un reingreso, que efectivamente se trate de un exempleado.

get /movimientosPersonal

Obtener la información básica de todos los movimientos de personal existentes.

Secured by oauth_2_0
  • Request
  • Response

Query Parameters

  • pagina: (number )

    Opcional. Numero de página a regresar.

  • tamanoPagina: (number- default: 20 )

    Opcional. Tamaño de la página a regresar. Solo es válido si se especifica el parámetro "pagina".

  • orden: (string- default: id )

    Opcional. Nombre del campo que será el criterio para realizar el paginado de los resultados. Sólo es válido si se especifica el parámetro "pagina"

  • tipoMovimiento: (one of Baja,Reingreso,Cambio lateral,Corrección,Democión,Promoción,Transferencia )

    Filtrado de movimientos de personal por tipo.

  • fechaInicial: (string )

    Filtrado de movimientos de personal por fecha (todos los movimientos aplicados a partir de esta fecha).

  • fechaFinal: (string )

    Filtrado de movimientos de personal por fecha (todos los movimientos aplicados hasta esta fecha).

HTTP status code 403

No se tienen los permisos necesarios para crear movimientos de personal.

Body

Type: application/json

Example:

{
  "error": {
    "code": 403,
    "message": "No se tienen los permisos necesarios para crear movimientos de personal."
  }
}

HTTP status code 422

Error de validación en los datos del movimiento de personal. Puede ocurrir si alguno de los datos obligatorios no fue especificado, el empleado no existe, el tipo de movimiento no es correcto, la fecha no es aplicable de acuerdo al tipo de movimiento, la nueva posición no refleja el tipo de movimiento especificado, la posición no tiene habilitado el tipo de plantilla que se quiere asignar o no se especificó el motivo de baja (para el tipo de movimiento "Baja").

Body

Type: application/json

Example:

{
  "error": {
    "code": 422,
    "message": "Error de validación en los datos del movimiento de personal."
  }
}
