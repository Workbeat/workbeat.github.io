/empleados get post

get

Obtener la información básica de todos los empleados existentes.

post

Crear un empleado (asignándole una posición obligatoriamente. Opcionalmente se pueden asignar atributos al empleado).

Es requerido tener el permiso para crear empleados.

Se valida:

  • Que se especifiquen los siguientes campos: nombre, apellido paterno, fecha de nacimiento, ID de la posición, tipo de plantilla y fecha de ingreso.
  • Que todos los campos alfanuméricos tengan la longitud máxima permitida.
  • Que la posición que se quiere asignar al empleado exista.
  • Que el tipo de plantilla que se quiere asignar al empleado esté habilitado para la posición.
  • Que no exista otro empleado cuyo RFC, CURP o número de seguro social sea el mismo.
  • Que cada uno de los atributos de empleado que se quieren asignar existan.

get /empleados

Obtener la información básica de todos los empleados existentes.

Secured by oauth_2_0
  • Request
  • Response

Query Parameters

  • pagina: (number )

    Opcional. Numero de página a regresar.

  • tamanoPagina: (number- default: 20 )

    Opcional. Tamaño de la página a regresar. Solo es válido si se especifica el parámetro "pagina".

  • orden: (string- default: id )

    Opcional. Nombre del campo que será el criterio para realizar el paginado de los resultados. Sólo es válido si se especifica el parámetro "pagina"

  • filtro: (string )

    Filtrado de empleados por nombre, apellidos, correo electrónico, posición o atributo default de la posición (usualmente "Departamento"). Si no se especifica regresa el listado completo de empleados.

  • incluirBajas: (string )

    Si se incluye entonces el resultado incluirá a los exempleados de la empresa. Si no se especifica, el resultado incluirá solamente empleados activos.

HTTP status code 422

Error de validación en los datos del empleado. Puede ocurrir si alguno de los datos obligatorios no fue especificado, alguno de los campos alfanuméricos exceda la longitud máxima, no existe la posición que se quiere asignar al empleado, la posición no tiene habilitado el tipo de plantilla que se quiere asignar al empleado, existe otro empleado con el mismo RFC, CURP o número de seguro social o alguno de los atributos de empleado que se le quieren asignar no existen.

Body

Type: application/json

Example:

{
  "error": {
    "code": 422,
    "message": "Error de validación en los datos del empleado."
  }
}
